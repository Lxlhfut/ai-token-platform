## 改造：扫码充值 + 兑换码按 Token 量生成 + 管理端修复

### 背景
1. 管理员「生成兑换码」按钮无响应（排查后确认函数存在，但增强为支持按 Token 量和金额两种模式）
2. 需要用户自主微信/支付宝扫码充值（模拟支付流程）
3. 兑换码生成需要支持按 Token 量或金额

### 变更内容

**1. models.py**
- 新增 `RechargeOrder` 表（充值订单），含 order_no、user_id、amount、pay_method（wechat/alipay）、status（pending/paid/cancelled）、paid_at
- 新增 `RechargeOrderStatus` 枚举

**2. schemas.py**
- 新增 `RechargeOrderCreate`（amount + pay_method）
- 新增 `RechargeOrderOut`

**3. routers/user.py**
- 新增 `POST /api/user/recharge-order` - 创建扫码充值订单
- 新增 `POST /api/user/recharge-order/{order_no}/confirm` - 模拟支付成功回调（用户点「我已支付」）
- 新增 `GET /api/user/recharge-orders` - 用户查看自己的充值订单
- 充值后自动记录 Transaction，调用 `recharge_balance`

**4. routers/admin.py**
- 新增 `GET /api/admin/recharge-orders` - 管理员查看所有充值订单

**5. dashboard.html - 充值卡片大改造**
- 新增「扫码充值」和「兑换码充值」两个 Tab 切换
- 扫码充值面板：微信/支付宝切换 + 金额输入 + 快捷金额（5/10/20/50/100）+ 生成支付二维码
- 支付弹窗：显示二维码占位区域 + 订单号 +「我已支付完成」按钮
- 充值记录表格（显示所有订单的时间/金额/方式/状态）

**6. admin.html**
- 新增「充值订单」管理区域，显示所有订单

**7. app.js**
- 新增 `switchRechargeTab(tab)` - 充值方式 Tab 切换
- 新增 `selectPayMethod(method)` - 选择微信/支付宝
- 新增 `setScanAmount(amount)` - 快捷金额
- 新增 `closePaymentModal()` / `confirmPayment()` - 支付弹窗管理
- 新增 `loadUserPanel` 中加载充值订单列表
- `showRedeemForm` 改造：支持「按金额」和「按 Token 量」两种模式切换（`switchRedeemMode`），Token 量模式自动按 0.004元/1K tokens 换算金额
- `loadAdminPanel` 中加载充值订单列表

**8. style.css**
- 新增 `.payment-modal`、`.payment-content`、`.qr-placeholder`、`.qr-inner` 样式

### 支付流程说明
1. 用户选支付方式 + 金额 → 点「生成支付二维码」
2. 弹出模拟二维码 + 订单号
3. 用户点「我已支付完成」→ 后端标记 paid + 充值到余额
4. 余额立即更新，可在 API Key 调用中使用

### 注意
- 扫码支付为模拟模式（无真实微信/支付宝接入），适合本地测试
- 兑换码 Token 量换算公式：amount = (tokens / 1000) * 0.004，保留两位小数
