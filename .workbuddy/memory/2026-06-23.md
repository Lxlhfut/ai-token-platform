## Bug 排查与修复（多按钮无响应）

### 确认的 Bug 列表和修复

1. **`showTool()` / `switchRechargeTab()` 使用 `event.target`**（最严重）
   - `event` 全局变量在某些浏览器/严格模式下可能为 undefined，导致函数抛错、按钮无响应
   - `showTool` 还用了 `querySelectorAll('.tool-tab')` 不限作用域，点工具tab时误删充值tab的active状态
   - **修复**：改为传 `this` 参数，用 `btn.closest('.tool-tabs')` 限定作用域

2. **`payment-modal` 嵌套在 `recharge-scan` 内**
   - `recharge-scan` 被 `display:none` 时，`position:fixed` 的弹窗也会隐藏
   - **修复**：将弹窗移到 `recharge-scan` 外部（仍在充值卡片内）

3. **`confirmPayment()` 错误信息显示在弹窗后面**
   - 支付失败错误写入 `redeem-msg`，被弹窗遮住，用户看不到
   - **修复**：在弹窗内部加 `<p id="pay-error-msg">` 直接展示错误，并添加按钮 loading 状态

4. **管理员生成兑换码 alert 双 `¥` 问题**
   - `'¥' + fmtMoney(amount)` = `¥¥xxx.xxxx`，`fmtMoney` 本身已含 `¥`
   - **修复**：去掉多余的 `¥` 前缀

5. **`deleteChannel` 无 try/catch**
   - `loadAdminPanel()` 失败时无提示
   - **修复**：加 try/catch + `alert` 提示

6. **`loadAdminPanel` 中 channels/pricing/users/usage 无独立 try/catch**
   - 任一 API 失败导致整个函数抛出，`.catch(() => clearToken())` 强制退出登录
   - **修复**：每个区段独立 try/catch，失败时显示"加载失败"提示

7. **`admin.py` `toggle_user` 错误信息写错**
   - "模型定价已存在" → "用户不存在"；"渠道不存在" → "不能禁用管理员账号"
   - **修复**：更正错误信息

## Session 3：API 调用 404 问题排查与修复

用户反馈 curl 调用 `/v1/chat/completions` 返回 404。

### 根因
**proxy.py 传给上游的 path 带有 `/v1` 前缀，而渠道 base_url 也以 `/v1` 结尾，导致双重 `/v1/v1/`：**
```
base_url: https://dashscope.aliyuncs.com/compatible-mode/v1
path:     /v1/chat/completions  ← 从路由路径直接传入
url:      https://dashscope.aliyuncs.com/compatible-mode/v1/v1/chat/completions  ❌
```
上游 DashScope 返回 404，代理透传回客户端。

### 修复
1. **`app/routers/proxy.py`**：所有传给 `proxy_openai_request` 的 path 去掉 `/v1` 前缀
   - `/v1/chat/completions` → `/chat/completions`
   - `/v1/completions` → `/completions`
   - `/v1/embeddings` → `/embeddings`
2. 添加了 `request.json()` 异常捕获（无 body 时返回 400 而非崩溃）
3. DeepSeek 渠道 models 限定为 `deepseek-chat,deepseek-reasoner`（避免 `qwen-turbo` 被随机路由到不支持它的渠道）

### 验证结果
```
Status: 200
Model: qwen-turbo
Reply: Hello! How can I assist you today?
Tokens: prompt=14, completion=11, total=25
```
用户 123456 余额从 ¥25.00 → ¥24.96，扣费 ¥0.0378，计费正常。
