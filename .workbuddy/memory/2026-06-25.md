## Bug 修复：代理端提现金额校验失败（同日第八次）

### 问题
代理端输入金额后点击提现 → 确认弹窗中点击「使用此收款码提现」→ 仍然报错「请输入正确的提现金额」。

### 根因
`confirmWithdrawWithBound()` 函数先调用了 `closeWithdrawConfirmModal()`（将 `_wdPendingAmount` 重置为 `0`），再用已被清零的变量调用 `doWithdraw()`，导致 `doWithdraw` 收到的金额始终为 0，触发 `amount <= 0` 校验失败。

### 修复
在 `confirmWithdrawWithBound()` 中，先保存 `_wdPendingAmount` / `_wdPendingPayMethod` / `window._boundQrcodePath` 到局部变量，再关闭弹窗并传局部变量调用 `doWithdraw()`。

## 固定金额扫码充值 + 管理员审核系统（同日第十次）

### 问题
平台使用个人微信/支付宝静态收款码，用户可任意输入金额，系统无法校验实际支付金额是否与系统记录的金额一致。

### 方案：固定面额收款码 + 管理员审核
用户只能选择预设面额（2/5/10/20/50/100/500），每个面额对应一张独立收款码（微信+支付宝各7张，共14张）。
微信/支付宝里对固定金额生成的收款码，扫码后金额被锁定，用户无法修改。管理员在后台核对账单后手动确认到账。

### 流程
1. 管理员上传各面额收款码（微信+支付宝）
2. 用户选择面额 → 系统展示对应收款码 → 扫码支付
3. 用户点击「我已支付」→ 订单进入「待审核」状态
4. 管理员核对微信/支付宝账单 → 确认到账（系统自动充值）或拒绝

## Windows CET 兼容性问题（同日第十二次）

**现象**：PowerShell 中运行 Python 直接崩溃，报错 `Fatal error. Your Windows doesn't fully support CET.`，连 `python --version` 都失败。

**原因**：Windows + Python 3.9 的 CET（Control-flow Enforcement Technology）兼容性问题，与代码无关。

**绕过方案**：通过 cmd 运行 Python（如 `run.bat`），不要用 PowerShell。PowerShell 中可用 `cmd /c "python ..."` 包裹。服务端代码和应用路由均正常。
